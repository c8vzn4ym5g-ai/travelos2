# Coding Rules

- Keep changes scoped to the active task.
- Prefer existing project patterns over new abstractions.
- Use TypeScript strictly.
- Do not introduce database, authentication, storage, or AI providers before the task list calls for them.
- Build usable product screens, not marketing pages.
- Use responsive layouts from the start.
- Treat private data as the default.

## Task Workflow

1. Read the task and relevant docs.
2. Make the smallest complete change.
3. Run typecheck, lint, or build when available and relevant.
4. Update `docs/Tasks.md` status.
5. Stop for review.
